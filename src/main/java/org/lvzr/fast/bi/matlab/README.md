
Python 与 Matlab 哪一个对量化投资和分析更有帮助？
https://www.zhihu.com/question/29186103/answer/43496781

我用Matlab，主要因为Matlab在学术相关的机器学习领域应用广泛, 社区强大。技术上讲，任何Matlab可以完成的任务都可以用R/python完成
, 用Matlab或R完全取决于你所应用的领域，另外你应该把开发时间也考虑进去, 因为当你开发一个复杂的交易策略时，建模阶段是最耗时的。在
我所从事的领域，Matlab应用很广，已经积累了大量的可直接使用的代码，R/python根本无法比拟(我用过一段时间的R)，这将大大缩短建模时间
，尤其是当我需要快速建模的时候，所以Matlab决对是我的首选，当我执行策略时我可以选C++或Matlab, 

取决于策略的类型，我当然不会用Matlab执行HFT策略。的确，统计学术界用R很多，但在工程和机器学习领域Matlab用户群更大。当你选开发语言
时不要一味从技术的角度出发，有时从商业角度看问题，你会做出不同的选择。对我而言时间就是金钱，这些节约的开发时间要是算成人力成本的
话肯定多于购买Matlab的花费。你当然可以选R/python，如果你用到很多统计学的包，这是个人喜好问题，没啥好争的。你要是一定想知道有啥事matlab
能轻松完成而python/R不行的，我给你举个例子，Matlab可以直接发信号给broker交易，比如Bloomberg EMSX，X_TRADER  和Interactive Brokers。
这意味着我用Matlab开发的模型不用改就可用于中低频交易，R在这点上要差很多。另外，Matlab在并行计算领域比R有优势。还有一点，Matlab有很
多第三方工具可用。
